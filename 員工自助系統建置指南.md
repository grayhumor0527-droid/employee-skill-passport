## 建置步驟

### Phase 1：基礎建置（1-2 天）

#### Step 1：建立 Google Sheets

1. 開啟 Google Drive
2. 新增 → Google 試算表
3. 命名為「摩研員工主檔」
4. 匯入 `員工主檔範本.csv`

**工作表結構**：

| 工作表名稱 | 用途 |
|-----------|------|
| 員工主檔 | 所有員工基本資料 |
| 薪資紀錄 | 歷史薪資明細 |
| 考核申請 | 員工提交的考核申請 |
| 薪資確認 | 員工確認薪資的紀錄 |

#### Step 2：設定員工主檔欄位

| 欄位 | 資料類型 | 說明 | 必填 |
|------|----------|------|:----:|
| A | 員工編號 | CA111-001 格式 | ✓ |
| B | 姓名 | 中文姓名 | ✓ |
| C | 手機號碼 | 登入用 | ✓ |
| D | 身分證字號 | 加密儲存 | |
| E | 出生日期 | YYYY-MM-DD | |
| F | 到職日期 | YYYY-MM-DD | ✓ |
| G | 職級 | 1-6 | ✓ |
| H | 職稱 | 學徒/半技/師傅/領班 | ✓ |
| I | 技能分數 | 0-431 | ✓ |
| J | 日薪 | 數字 | ✓ |
| K | 外地加給 | 日薪 × 0.125 | ✓ |
| L | 指導導師 | 師傅姓名 | |
| M | 會開貨車 | 是/否 | |
| N | 緊急聯絡人 | 姓名/關係/電話 | |
| O | 保險啟動日 | YYYY-MM-DD | |
| P | 狀態 | 在職/離職/外調 | ✓ |
| Q | 技術能力 | 1-5 | |
| R | 工作態度 | 1-5 | |
| S | 出勤狀況 | 1-5 | |
| T | 團隊合作 | 1-5 | |
| U | 學習成長 | 1-5 | |
| V | 考核總分 | 公式自動計算 | |
| W | 考核等第 | 公式自動判定 | |
| X | 最近考核日 | YYYY-MM-DD | |
| Y | 備註 | 自由填寫 | |

#### Step 3：設定公式

**V 欄（考核總分）**：
```
=IF(Q2="","",ROUND(Q2*0.4+R2*0.2+S2*0.15+T2*0.15+U2*0.1,1))
```

**W 欄（考核等第）**：
```
=IF(V2="","",IF(V2>=4.5,"優等",IF(V2>=4,"甲等",IF(V2>=3,"乙等",IF(V2>=2,"丙等","丁等")))))
```

**K 欄（外地加給）**：
```
=ROUND(J2*0.125,0)
```

### Phase 2：部署 Apps Script（1 天）

#### Step 1：開啟 Apps Script

1. 在 Google Sheets 中
2. 擴充功能 → Apps Script
3. 刪除預設程式碼
4. 貼上 `Google_Apps_Script.js` 內容

#### Step 2：設定 CONFIG

```javascript
const CONFIG = {
  SHEET_EMPLOYEES: '員工主檔',
  SHEET_SALARY: '薪資紀錄',
  SHEET_APPLICATIONS: '考核申請',
  SHEET_CONFIRMATIONS: '薪資確認',

  // 填入您的 LINE Notify Token
  LINE_NOTIFY_TOKEN: 'YOUR_TOKEN_HERE',

  // 填入管理者 Email
  ADMIN_EMAIL: 'your-email@example.com',

  APPLICATION_COOLDOWN_DAYS: 30
};
```

#### Step 3：部署為網頁應用程式

1. 點擊「部署」→「新增部署作業」
2. 類型選擇「網頁應用程式」
3. 執行身分：「我」
4. 存取權：「任何人」
5. 點擊「部署」
6. 複製網頁應用程式網址

#### Step 4：設定觸發器

1. 左側選單 → 觸發條件
2. 新增觸發條件
3. 函式：`dailyCheck`
4. 事件來源：時間驅動
5. 類型：每日計時器
6. 時間：上午 8-9 點

---

### Phase 3：設定 Email 通知（10 分鐘）

> 注意：LINE Notify 已於 2025年3月31日 停止服務，改用 Email 通知

#### Step 1：修改 CONFIG

將 `ADMIN_EMAIL` 改成您的 Email：
```javascript
ADMIN_EMAIL: 'your-email@gmail.com',  // ← 改成您的 Email
```

#### Step 2：測試通知

1. 在 Apps Script 編輯器中
2. 選擇函式：`testEmail`
3. 點擊「執行」
4. 首次執行需授權 → 審查權限 → 選擇帳號 → 允許
5. 檢查您的 Email 是否收到測試信

#### Step 3：通知類型

| 事件 | Email 主旨 |
|------|-----------|
| 考核申請 | 【摩研】考核申請 - 員工姓名 |
| 薪資確認 | 【摩研】薪資確認 - 員工姓名 |
| 保險提醒 | 【摩研】保險啟動提醒 |

---

### Phase 4：上傳員工入口（30 分鐘）

1. 前往 https://sites.google.com/
2. 建立新網站
3. 命名為「摩研員工入口」
4. 插入 → 嵌入 → 嵌入程式碼
5. 貼上 `員工自助入口.html` 的內容
6. 發布網站
7. 設定存取權限（僅限特定人員）


### Phase 5：整合測試（1 天）

#### 測試清單

| 項目 | 測試內容 | 預期結果 |
|------|----------|----------|
| 登入 | 輸入手機號碼/員工編號 | 顯示員工資料 |
| 職等查看 | 點擊「我的職等」 | 顯示技能分數、升級進度 |
| 薪資確認 | 點擊「薪資確認」 | 顯示薪資明細、可確認 |
| 考核申請 | 填寫申請表單 | 提交成功、主管收到通知 |
| LINE 通知 | 提交申請 | LINE 群組收到通知 |

## 日常維護

### 資料同步流程

```
Google Sheets（員工主檔）
         ↓ 手動匯出 JSON
員工入口 HTML（更新 employeeData）
         ↓
人事資料卡 v2（保持同步）
```

### 每週維護事項

| 事項 | 頻率 | 負責人 |
|------|------|--------|
| 同步出勤資料 | 每日 | 自動（sync_sheets.py） |
| 更新薪資紀錄 | 每期 | 會計 |
| 審核考核申請 | 有申請時 | 主管 |
| 備份 Google Sheets | 每週 | 自動 |

### 常見問題

**Q：員工無法登入？**
A：確認手機號碼是否正確填入 Google Sheets

**Q：薪資資料沒有更新？**
A：需手動將薪資紀錄加入 Google Sheets

**Q：LINE 通知沒收到？**
A：檢查 Token 是否正確、機器人是否已加入群組
